Pair programming

Pair programming is a way of programming together with one computer. The first one (the driver), writes the code and the second one (the navigator), reviewes the copde as it is being written.

This method is used in agile software development and can be done in person or remote using shared screens or tools to edit the code simultaneously.

During the coding process the driver focus on the task at hand using the navigator as a guide. The navigator main focus is to plan, strategize, improve and solve problems that might be ahead.

Some pros to pair programming

  • Shorter code with fewer bugs and better design. Pair programming reduces bugs by 15 to 20 % according to studies.
  • Increased code quality when sharing best practices. Pair programming leads to more robust soulutions.
  • Sharing knowledge!
